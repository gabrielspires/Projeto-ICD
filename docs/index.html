Introdução¶

Tema¶

Dota 2 um jogo eletrnico do gnero Multiplayer Online Battle Arena (MOBA, "arena de batalha multijogador online"), tambm conhecido como Action Real-Time Strategy (ARTS, "estratgia de ao em tempo real") onde o jogador controla um personagem em uma batalha entre dois times, cujo objetivo derrotar a equipe adversria e, para isso, , indispensavelmente, necessrio destruir o Ancestral (Ancient), que, por sua vez, se localiza no centro da base inimiga. Cada base defendida por torres e ondas de unidades (chamadas de creeps) que percorrem os caminhos principais do mapa (chamadas de trilhas, ou lanes) que levam suas bases. Existem 3 trilhas no mapa do jogo, que por conveno so chamadas de trilha de cima (Top, atravessa a parte de cima do mapa), trilha do meio (Mid ou Middle, caminho que atravessa o centro do mapa) e trilha de baixo (Bot ou Bottom, caminho que atravessa a parte de baixo do mapa).

Na imagem abaixo podemos ver o minimapa do Dota2 e ao lado um diagrama representando o layout de um jogo do gnero MOBA, onde os pontos roxos e rosas representam as torres de cada time, com exceo do maior ponto localizado nos cantos da imagem dentro de cada base, que representa o Ancient.

Descrição da base¶

Os dados foram coletados utilizando a API no oficial do jogo, chamada OpenDoTA, nela coletamos dados de vrias partidas do ano de 2015, assim como os dados dos jogadores presentes em cada partida e todas as suas aes, como por exemplo as coordenadas das posies que o jogador passou durante a partida, o heri que escolheu, quantas vezes morreu, entre outros. Ns optamos por utilizar os mesmos dados que usamos no trbalho final de Introduo a Banco de dados, mas os dados coletados foram disponibilizados em formato json, e no em formato de tabelas como precisvamos, ento foi preciso selecionar os dados que queramos entre aqueles fornecidos pela API e montar um banco de dados a partir deles.

Uma caracterstica do formato json usar o tipo abstrato de dados dicionrio, que composto de pares de chaves e valores, onde cada chave pode conter uma lista de outras chaves e seus valores, isso faz com que existam vrias "sub-tabelas" nos dados, o que impossibilita a importao dos dados diretamente para um banco de dados convencional. Dessa forma, foram criadas tabelas separadas para cada conjunto de valores do json, sendo que cada tabela referencia o seu "pai" original atravs de chaves estrangeiras, dessa forma conseguimos manter a relao dos dados como era originalmente feito no json.

Um exemplo dos dados pode ser visto na imagem abaixo.
Figura 2: Amostra dos dados coletados em formato json

Alguns dados no podiam ser buscados atravs da API, como por exemplo os nomes e nmeros identificadores de cada personagem ou item do jogo, que so informaes constantes, que no mudam entre as partidas, mas precisvamos dessas informaes uma vez que a maior parte dos dados numrica, e no seria interessante ter apenas nmeros, mas sim nomes e significados para que os dados fizessem mais sentido. Por fim encontramos esses dados tambm em formato json, compatveis com os dados da API, o que fez com que ficasse mais natural a relao entre as informaes.

Para condensar os dados em um nico dataframe, para ajudar em algumas anlises, precisamos encontrar uma forma de unir as tabelas. Sabemos que em cada partida, existem 10 jogadores, 5 em cada um dos dois times, j que vamos ignorar as partidas com bots, ento uma maneira de condensar os dados somando as estatsticas de cada time e adicionar essas estatsticas nas linhas de cada partida, separando as partidas por time, ou seja, onde antes existia uma entrada por partida, agora ir existir uma entrada por time para cada uma das partidas.

df = pd.read_sql_query('''
SELECT *
FROM match AS m
INNER JOIN
    (SELECT player_match_id,REPLACE(REPLACE(sum(player_slot),10,"Radiant"),650,"Dire") Team,sum(gold_spent),sum(gold),sum(xp_per_min),sum(level),sum(hero_healing),sum(hero_damage),sum(tower_damage),sum(last_hits),sum(kills),sum(denies),sum(deaths),sum(gold_per_min),sum(assists)
    FROM player
    WHERE player_slot < 5
    GROUP BY player_match_id
    UNION
    SELECT player_match_id,REPLACE(REPLACE(sum(player_slot),10,"Radiant"),650,"Dire") Team,sum(gold_spent),sum(gold),sum(xp_per_min),sum(level),sum(hero_healing),sum(hero_damage),sum(tower_damage),sum(last_hits),sum(kills),sum(denies),sum(deaths),sum(gold_per_min),sum(assists)
    FROM player
    WHERE player_slot > 5
    GROUP BY player_match_id) AS p
ON m.match_id = p.player_match_id
INNER JOIN
    (SELECT id game_mode_id, name game_mode_name
    FROM game_mode
    ) as gm
ON m.game_mode = gm.game_mode_id
INNER JOIN
    (SELECT id lobby_type_id, name lobby_type_name
    FROM lobby_type 
    ) as lt
ON m.lobby_type = lt.lobby_type_id
WHERE human_players = 10
''', conn)
